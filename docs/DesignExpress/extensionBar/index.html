ExtensionBar

디자인 익스프레스는 기존 Github에서 프로젝트를 관리하는 방식과 유사하게 프로젝트들을 관리합니다. 프로젝트는 여러 사람의 협업을 통해 완성됩니다. 이 때 각각 사람들의 작업영역이 겹치면 프로젝트의 충돌이 생길 수 있습니다. 이를 방지하기 위해 브랜치라는 개념을 도입합니다. 브랜치란 독립적으로 어떤 작업을 진행하기 위한 개념입니다. 필요에 의해 만들어지는 각각의 브랜치는 다른 브랜치의 영향을 받지 않기 때문에, 여러 작업을 동시에 진행할 수 있습니다. 또한 이렇게 만들어진 브랜치는 다른 브랜치와 병합(Merge)함으로써, 작업한 내용을 다시 새로운 하나의 브랜치로 모을 수 있습니다. ExtensionBar UI 버튼들을 통해 프로젝트 관리하는 법을 소개합니다.

깃에 대해서

TopBar#

  • TopBar 구성: 프로젝트 이름/오너 이름 // 브랜치 이름 버튼

  • 브랜치 이름을 클릭하면 현재 프로젝트의 브랜치 관리창(Revisions)이 열립니다.

  • 을 클릭하면 현재 브랜치를 서버와 동기화합니다. (이 버튼을 클릭하기 전 브랜치 관리를 통해 커밋(저장) 먼저 수행되어야 합니다.)
danger

두 명이상의 유저가 같은 브랜치에서 작업 시 깃이 엉킬 위험이 있습니다. 꼭 자신만의 브랜치를 생성해주세요

Revisions#

  • 버튼 클릭하면 새로운 브랜치 생성창이 열림
  • 버튼 클릭하면 삭제할 브랜치 입력창이 열림(현재 선택된 브랜치는 삭제할 수 없습니다. 현재 브랜치를 삭제하고 싶으면 브랜치를 전환해야 합니다.)
  • 버튼 클릭하면 서버와 통신해 최신 상태의 브랜치를 받아옴

SideBar#

  • 버튼 클릭 시 현재 프로젝트 상황을 로컬 브라우저에 저장합니다. 서버에 저장이 아닌 임시 저장입니다.
danger

클리너 프로그램등의 사용 등으로 브라우저의 쿠키나 로컬 스토러지가 정리될 시 브라우저에 저장된 데이터는 소멸될 수 있습니다.

  • 클릭 시 브랜치 커밋(저장)/로그 관리를 할 수 있는 창으로 진입합니다.
  • WorkSpace 창으로 진입합니다.
  • 메세지 창을 띄어줍니다.

브랜치 관리#

;

  • 변경점이 생기면 changed 탭을 통해 커밋(저장)을 할 수 있습니다.
  • 로그탭을 클릭하면 현재 브랜치의 로그(history)를 확인할 수 있습니다.
  • 로그탭에서 과거 History 링크를 클릭하면 과거 작업 환경으로 돌아갈 수 있습니다.
  • 과거 작업 환경에서 새로운 작업을 진행할려면 Revisions 창에 들어가서 새로운 브랜치를 만들어서 작업을 시작해야합니다.

WorkSpace#

;

  • WorkSpace 탭에서 현재 작업 환경을 저장할 수 있습니다. 여기서 말하는 작업 환경은 작업 탭들(ex. newInspector, graphic 등)의 구성을 말합니다.
    Local은 현재 프로젝트에서만 적용이 됩니다.
    Global은 전체 프로젝트에서 적용이 됩니다.

  • Windows 탭에서 사용할 탭들을 선택할 수 있습니다.
